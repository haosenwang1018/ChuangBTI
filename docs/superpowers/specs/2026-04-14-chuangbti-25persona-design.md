## 2. 引擎设计

### 2.1 轴定义（6 根双极轴）

| 轴 ID | 负端 (-) | 正端 (+) | 含义 |
|---|---|---|---|
| ACT | 思考/等待 | 执行/上线 | 行动倾向 |
| RISK | 稳健/保守 | 激进/全押 | 风险偏好 |
| TEAM | 单干/独立 | 控场/领导 | 协作方式 |
| AI | 低依赖 | 高依赖 | AI 浓度 |
| SHOW | 低调务实 | 聚光灯 | 叙事欲 |
| DRIVE | 内驱（热爱/使命） | 外驱（证明/财富） | 驱动源 |

### 2.2 打分规则

- 每根轴 4 题
- 每题 3 选项，分值 `-1 / 0 / +1`
- 4 题累加 → 轴得分范围 `-4 ~ +4`
- 用户最终向量：`U = [act, risk, team, ai, show, drive]`

### 2.3 匹配算法

```
对每个人格 P (除 FLEX):
    d = sqrt(sum((U_i - P_i)^2 for i in 6 axes))
winner = argmin(d)
maxDistance = sqrt(6 * 8^2) ≈ 19.596
similarity = round(100 * (1 - d / maxDistance))
```

### 2.4 FLEX 特殊触发

**规则**：如果用户 6 根轴的得分绝对值**全部 ≤ 1**（即每根轴都反复横跳、没明显偏向），判为 FLEX 变色龙。

伪码：
```js
if (axes.every(v => Math.abs(v) <= 1)) return FLEX
```

FLEX 不参与距离匹配。

## 4. 24 题题目结构

每根轴 4 题 × 6 = 24 题，每题 3 选项（-1 / 0 / +1）。

**出题 checklist**：

| 轴 | 4 题要覆盖的场景 |
|---|---|
| ACT | ① 产品上线策略 ② 看到机会的反应 ③ 计划 vs 执行 ④ 复盘 vs 迭代 |
| RISK | ① 要不要 all in ② 现金流策略 ③ 辞职/保底 ④ 遇到黑天鹅 |
| TEAM | ① 团队规模偏好 ② 决策方式 ③ 联创依赖度 ④ 招第一个人的标准 |
| AI | ① 写文案怎么做 ② 代码怎么写 ③ 重复劳动处理 ④ 对 AI 的态度 |
| SHOW | ① 朋友圈/社媒发不发 ② 路演心态 ③ 被采访反应 ④ 成就归因 |
| DRIVE | ① 为啥创业 ② 深夜动力来源 ③ 想象成功画面 ④ 失败后怎么办 |

**风格约束**：情景化、带梗、创投语境（BP/路演/飞书/TS/联创），**不涉及酒/饭局**（原"敬局彩蛋"机制已砍）。

**具体题干**：在 writing-plans 阶段逐条落地。

## 5. 结果页布局（轻量版）

**顺序**（顶→底）：

1. Kicker 小字：`你的创始人人格`
2. **人格大图**：`public/personas/{CODE}.png`，居中，max-width 320px

4. 金句 intro（斜体灰字）

8. 按钮：`下载分享卡` / `复制部署命令`

**相比原版砍掉**：TOP 5 列表、匹配度百分比 badge、雷达图、kicker 的多变体（drunk/fallback）、secondary 人格展示。
